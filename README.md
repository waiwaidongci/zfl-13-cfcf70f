# 潮汐池生态模拟

静态浏览器小游戏，直接打开 `index.html` 即可运行。

玩家通过放置岩缝、海藻、贝群和遮阴来维持潮间带生态稳定。系统包含潮汐循环、物种数量变化、捕食关系、低水位压力、生态稳定度评分和生态预算机制。

## 核心测试

核心计算逻辑已从浏览器交互代码中拆分，位于 `core.js`。可通过 Node.js 运行轻量测试：

```bash
node test.js
```

测试覆盖以下核心模块：

- **潮位阶段判断** (`phaseName`)：满潮、涨潮、退潮、低潮四个阶段的边界值判断
- **生态稳定度评分** (`stabilityScore`)：庇护所、食物、拥挤惩罚、物种平衡对分数的影响
- **物种统计** (`totals`)：空网格、单格、多格的物种数量累加
- **邻格查找** (`neighbors`)：中间格、角落格、边缘格的邻居数量与坐标验证
- **潮位计算** (`tideLevel`)：潮位波动范围与初始值验证
- **深拷贝网格** (`deepCloneGrid`)：深拷贝独立性验证

## 生态预算系统

每次放置或移除调控元素都会消耗或返还预算，初始预算为 40。右侧面板顶部显示当前剩余预算与进度条，预算低于 10 时进度条变红警示。

各工具的成本（放置消耗 / 移除回收）：

| 工具 | 成本 | 说明 |
|------|------|------|
| 岩缝 | 3 | 为生物提供遮蔽，提升稳定度 |
| 海藻 | 2 | 提供食物来源，吸引螺类 |
| 贝群 | 4 | 提供养分，吸引海星捕食 |
| 遮阴 | 1 | 减缓低潮时的生物压力 |
| 清理 | 回收 | 移除格子上所有元素并按原价返还预算 |

**预算不足时**放置操作会被阻止，画布下方会弹出红色提示「预算不足！」。合理规划放置顺序、及时移除不需要的元素回收预算是重要策略。

## 任务挑战模式

点击顶栏右上角的「任务挑战」按钮即可进入挑战选择界面。当前提供三个挑战：

- **稳定守护者**：在第6天结束前，将生态稳定度维持在80以上。
- **贝类平衡师**：在第5天到第8天期间，贝类数量保持在18~28之间。
- **低潮抗压**：在10天内，累计低潮压力事件不超过6次。

进入挑战后，右侧面板顶部会出现挑战进度区，实时展示各目标的完成状态（黄色待完成、绿色已达成、红色失败）。挑战完成或失败时，画布上方会弹出状态提示。

**重置按钮行为**：在挑战中点击「重置」会将潮汐池恢复到该挑战的初始状态（第1天、物种与工具配置归零、预算恢复至 40），而不会退出当前挑战。点击进度区底部的「退出挑战」可回到自由模式。

## Canvas 点击坐标验证说明

### 问题背景
在移动端或浏览器缩放后点击 Canvas 格子时，可能出现点击位置与实际选中格子不匹配的问题，尤其是画布被 CSS 拉伸后更明显。

### 修复原理
1. **坐标换算修正**：使用 `canvas.width / rect.width` 计算缩放比例，将 CSS 像素正确转换为 Canvas 实际像素坐标
2. **宽高比锁定**：CSS 中使用 `aspect-ratio: 920 / 620` 确保画布始终保持正确比例，避免变形
3. **响应式适配**：`max-width` 和 `max-height` 限制最大尺寸，`width: 100%` 确保小屏幕自适应

### 手动验证步骤

#### 桌面端验证
1. 打开 `index.html`，确保浏览器窗口宽度大于 980px（两栏布局）
2. 选择「岩缝」工具，点击画布左上角的第一个沙格子（第1行第2列，坐标约 x=90, y=80）
3. 验证：岩缝应准确出现在点击的格子上，不会偏移到相邻格子
4. 选择「清理」工具，点击刚才放置岩缝的格子
5. 验证：岩缝被正确清除，不会误清除其他格子

#### 浏览器缩放验证
1. 使用浏览器缩放功能（Ctrl+/- 或 Cmd+/-），分别测试 75%、100%、125%、150% 缩放比例
2. 在每个缩放比例下，点击画布四角的格子和中心格子
3. 验证：点击位置与选中格子始终一一对应，无偏移现象

#### 移动端/窄屏验证
1. 缩小浏览器窗口宽度至 640px 以下（移动端布局）
2. 选择「海藻」工具，依次点击第 1 行、第 5 行、第 9 行的中间格子
3. 验证：海藻准确放置在点击的格子中，不会出现横向或纵向偏移
4. 尝试点击画布边缘的格子（最左列、最右列、最上行、最下行）
5. 验证：边缘格子也能被正确选中，不会触发相邻格子

#### 窗口大小变化验证
1. 缓慢拖动浏览器窗口边缘，改变窗口宽度
2. 在拖动过程中停止，点击画布上的任意格子
3. 验证：窗口大小变化后，点击位置仍然准确对应正确的格子

### 验证通过标准
- 所有缩放比例下，点击位置与选中格子完全一致
- 桌面端和移动端布局下均无偏移
- 四角和边缘格子点击准确
- 快速点击和慢速点击均能正确响应
